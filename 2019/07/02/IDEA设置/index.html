IDEA个人设置
发表于 2019-07-02 | 更新于 2019-07-02
软件使用

注：个人用的IDEA是2019新版的，可能设置界面跟老版的略有不同

快速搜索设置选项：

更改注释字体的颜色

File | Settings | Editor | Color Scheme | Language Defaults

Comments目录下的

Block comment 块注释

Line comment 行注释

Doc Comment 类文件

快捷键配置

File | Settings | Keymap

Generate：构造器/Getter/Setter生成等

其他对应keymap：

智能提示

Show Intention Actions

打开文件所在目录

Show In Explorer

实现类

implementation

修改文件名

Rename

..

鼠标滚轮字体大小变化设置

File | Settings | Keymap | Editor Actions 下，找到 “Decrease Font Size”和“Increase Font Size”两个选项。分别对应缩小文字和放大文字。

IDEA全局文件编码设置

File | Settings | Editor | File Encodings

建议以上都选为UTF-8

主题设置

File | Settings | Appearance & Behavior | Appearance

界面字体设置

File | Settings | Appearance & Behavior | Appearance

注：同时也影响Messages栏的build栏提示文字（编译不通过会提示），字体不包括中文格式则不显示中文，不确定是否与编译器有关，如下

Maven相关配置

File | Settings | Build, Execution, Deployment | Build Tools | Maven

项目工程全局配置

File | Other Settings | Settings for New Projects

检查更新设置

File | Settings | Appearance & Behavior | System Settings | Updates

代码字体设置

File | Settings | Editor | Font

编译器配置

File | Settings | Build, Execution, Deployment | Compiler | Java Compiler

设置代码提示对大小写不敏感

File | Settings | Editor | General | Code Completion


Match case 取消勾选

2017版idea

File | Settings | Editor | General | Code Completion Case | Sensitive Completion = None

开启自动编译

File | Settings | Build, Execution, Deployment | Compiler

然后在打开的页面上快捷键ctrl + shift + a 搜索命令：registry –> 勾选compiler.automake.allow.when.app.running*

后期跑项目开启自动编译可以配合jrebel小火箭插件实现热部署

自动导包和import智能移除

File | Settings | Editor | General | Auto Import

隐藏文件结构中的配置文件

刚接触idea新建工程时，一般会出现xxx.idea，xxx.iml文件，这俩文件我们几乎不会去动它的，所以我们可以把它们隐藏起来

File | Settings | Editor | File Types 在末尾加上 .idea;.iml

收起文件注释

File | Settings | Editor | General | Code Folding

Documentation comments 勾选

快速一键打开全部注释，则单击鼠标右键，选择Folding -> Expand Doc comments

打开常用工具栏

具体步骤：顶部导航栏 - View -> 勾选 Toolbar & Tool Buttons
